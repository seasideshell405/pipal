---
name: todo_manager
description: 四象限任务管理技能。负责任务的创建、查看、完成、更新和删除。
tools:
  - create_todo
  - list_todos
  - complete_todo
  - update_todo
  - delete_todo
triggers:
  - 用户提到「任务」「待办」「四象限」
  - 用户要求创建/查看/完成/更新/删除任务

## Triggers

| 触发场景 | 行为 |
|---------|------|
| 用户说「帮我记个事」「建个任务」 | 提取信息→追问确认→调用 `create_todo` |
| 用户说「看看我的任务」「四象限」 | 调用 `list_todos(status: "pending")` → 按象限展示 |
| 用户说「已完成任务」「完成记录」 | 调用 `list_todos(status: "done")` → 展示 |
| 用户说「XX做完了」「划掉」 | 匹配任务 → 调用 `complete_todo` |
| 用户说「把XX改成…」「延期到…」 | 匹配任务 → 调用 `update_todo` |
| 用户说「把XX删掉」 | 匹配任务 → 调用 `delete_todo` |

## Steps

### 创建任务

#### Step 1: 提取信息

从用户语句中提取：

| 字段 | 提取规则 |
|------|---------|
| content | 任务内容 |
| importance | 用户明确说「重要/不重要」则记录；没说则追问 |
| urgency | 用户明确说「紧急/不紧急」则记录；没说则追问 |
| dueDate | 用户提到时间则提取为 YYYY-MM-DD；没提不加，不追问 |

#### Step 2: 追问确认

用以下模板向用户确认：

```
好的，已记录：
📌 任务：「{content}」
📂 象限：{重要性} · {紧急度}
{📅 截止日期：{dueDate}（如果有）}

{如果是不紧急且有截止日期}
要不要设置自动转化？离截止还剩 X 天时自动变成紧急任务提醒你？
阈值几天合适？（如 3 天、7 天）
```

**规则**：
- 重要/紧急没说明的 → 追问
- 截止日期没提 → 不加，不追问
- **自动转化（thresholdDays）**：仅当任务是 **不紧急** 且设了 **截止日期** 时，才询问是否需要。用户确认需要则追问阈值天数，说不用则不设
- **紧急任务** 不询问阈值

#### Step 3: 用户确认后 → 调用 `create_todo`

### 查看四象限

#### Step 1: 调用 `list_todos(status: "pending")`

#### Step 2: 按象限组织展示

按「重要·紧急」「重要·不紧急」「不重要·紧急」「不重要·不紧急」分组，忽略空象限。

每个任务标注动态紧急度（有截止日期+阈值且临近时自动提升为紧急）。

### 查看已完成任务

#### Step 1: 调用 `list_todos(status: "done")`

#### Step 2: 按完成时间倒序展示

### 完成任务

1. 先调用 `list_todos(status: "pending")` 获取任务列表
2. 根据用户描述的内容匹配到对应的任务，提取列表中该任务前面的短 id（中括号内，如 `a1b2c3d4`）
3. 调用 `complete_todo(id)` 标记完成（传短 id 即可，工具支持前缀匹配）
4. 回复确认 + 鼓励

### 更新任务

1. 先调用 `list_todos(status: "pending")` 获取任务列表
2. 根据用户描述的内容匹配到对应的任务，提取列表中该任务前面的短 id
3. 调用 `update_todo(id, ...)` 更新相应字段（传短 id 即可）
4. 回复更新后的状态

### 删除任务

1. 先调用 `list_todos(status: "pending")` 获取任务列表
2. 根据用户描述的内容匹配到对应的任务，提取列表中该任务前面的短 id
3. 调用 `delete_todo(id)` 删除（传短 id 即可）
4. 回复确认

## Templates

### 四象限展示模板

```
待办任务共 {count} 个

────────────────────────────────────
🔴 重要·紧急
  □ [{id前8位}] {任务内容}    截止: {dueDate} ⏰

────────────────────────────────────
🟡 重要·不紧急
  □ [{id前8位}] {任务内容}    截止: {dueDate}  → {thresholdDays}天前提醒

────────────────────────────────────
🟢 不重要·紧急
  □ [{id前8位}] {任务内容}

────────────────────────────────────
⚪ 不重要·不紧急
  □ [{id前8位}] {任务内容}

────────────────────────────────────
共 {count} 项待办
```

**规则**：
- 空的象限不展示
- 已自动变为紧急的标注提醒
- 没有截止日期的不显示日期行
- 每个任务前用 `□` 表示待办
- 每个任务带 `[{id前8位}]` 前缀，供后续操作（完成/更新/删除）提取 id 使用

### 已完成任务展示模板

```
✅ 已完成的任务

  ☑️ [{id前8位}] {任务内容}    完成于 {completedDate}

────────────────────
共 {count} 项已完成 🎉
```

**规则**：
- 按完成时间倒序排列
- 每个任务前用 `☑️`
- 每个任务带 `[{id前8位}]` 前缀
- 无已完成任务时显示：「还没有完成的任务，冲一个？💪」
