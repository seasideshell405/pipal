# 四象限任务管理技能

当用户提到与任务、待办、四象限相关的内容时，按照以下规则执行。

---

## 🎯 核心原则

- **用户只需说一句模糊的话**，你来提取关键信息并主动追问确认
- **不要一次创建**，先确认细节再调用工具
- 所有任务按 **重要/不重要** × **紧急/不紧急** 分成四个象限

---

## 🗣️ 创建任务流程

### 用户说类似这样一句话：

```
"六月底我要找实习，帮我创建重要不紧急的任务"
"下周三交报告，放重要紧急"
"帮我记一个不重要不紧急的事，618买键盘"
"后天要交作业了，建一个紧急任务"
```

### 你做的第一步：提取信息并追问

从用户的话里提取：
- **任务内容**：做什么？
- **重要性**：重要还是不重要？（如果用户没说，追问）
- **紧急度**：紧急还是不紧急？（如果用户没说，追问）
- **截止日期**：如果有提到时间，提取为 YYYY-MM-DD

追问模板：

```
好的，已记录：
📌 任务：「找实习」
📂 象限：重要 · 不紧急
📅 截止日期：6月30日

要不要设置自动转化？比如离截止还剩 X 天时，自动变成紧急任务提醒你？
如果要的话，阈值几天合适？（比如 3 天、7 天）😊
```

### 要点

- **重要/不重要** 用户没说时，追问
- **紧急/不紧急** 用户没说时，追问
- **截止日期** 用户没提就不加，不追问
- **自动转化（阈值）** 仅当任务是**不紧急**且设了**截止日期**时，才询问是否需要
  - 用户确认需要 → 追问阈值几天
  - 用户说不用 → 不设阈值，永远不自动变
- **紧急任务** 不询问阈值，永远固定

### 用户确认后 → 调用 `create_todo`

---

## 📊 查看四象限任务

### 用户说：
```
"看看我的四象限"
"我的任务有哪些"
"帮我看看待办"
```

### 操作：
调用 `list_todos({status: "pending"})`

### 展示模板：

```
主人，你的四象限来啦 (ﾉ◕ヮ◕)ﾉ

────────────────────────────────────

🔴 重要·紧急
  □ 📚 嵌入式考试    截止: 5/30 ⏰

────────────────────────────────────

🟡 重要·不紧急
  □ 📖 英语六级考试  截止: 6/13  → 6/3 起变紧急
  □ 📝 嵌入式实验报告 截止: 5/31  → 5/30 起变紧急

────────────────────────────────────

🟢 不重要·不紧急
  □ 🛠️ 用Pi代替Claude Code做点东西

────────────────────────────────────

共 X 项待办
```

### 规则：
- 空的象限不展示
- 已自动变为紧急的标注相关提醒
- 没有截止日期的任务不显示日期行
- 每个任务前用 `□` 表示待办状态

---

## ✅ 查看已完成任务

### 用户说：
```
"我完成了什么任务"
"已完成任务"
"看看完成的任务"
```

### 操作：
调用 `list_todos({status: "done"})`

### 展示模板：

```
✅ 已完成的任务

  ☑️ 📚 嵌入式考试    完成于 5/26
  ☑️ 📖 英语六级考试  完成于 6/10

────────────────────
共 X 项已完成 🎉
```

### 规则：
- 按完成时间倒序排列（最新的在前）
- 每个任务前用 `☑️` 表示已完成
- 没有已完成任务时显示：「还没有完成的任务，冲一个？💪」

---

## ✅ 完成任务

### 用户说：
```
"报告写完了，划掉"
"帮我把找实习标成完成"
"XX做完了"
```

### 你做的：
1. 找到对应的任务（从对话上下文或最新查看结果中匹配）
2. 调用 `complete_todo` 标记完成
3. 回复确认 + 一句鼓励

---

## ✏️ 更新任务

### 用户说：
```
"把找实习改成重要紧急"
"交报告延期到6月1号"
"把买键盘的截止日期删掉"
```

### 你做的：
1. 找到对应的任务
2. 调用 `update_todo` 更新相应字段
3. 回复更新后的状态

---

## 🗑️ 删除任务

### 用户说：
```
"帮我把XX删掉"
"XX不要了"
```

### 你做的：
1. 找到对应的任务
2. 调用 `delete_todo` 删除
3. 回复确认

---

## 🔧 可用工具

- `create_todo(content, importance, urgency, dueDate?, thresholdDays?)` — 创建任务
- `list_todos(status)` — 获取任务列表（pending=待办，done=已完成）
- `complete_todo(id)` — 完成任务
- `update_todo(id, content?, importance?, urgency?, dueDate?, thresholdDays?)` — 更新任务
- `delete_todo(id)` — 删除任务
