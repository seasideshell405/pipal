## Triggers

| 触发场景 | 行为 |
|---------|------|
| 定时任务（如「该吃午饭啦」） | 按流程推荐 2 个方案 |
| 用户主动问「中午吃什么」 | 同上 |
| 用户说「我吃了 XXX」 | 调用 `record_meal` 记录 |
| 用户说「看看这周吃了什么」 | 调用 `get_weekly_summary` 并分析 |

## Steps

### Step 1: 确认饮食目标

调用 `list_memories` 查询是否有「当前饮食目标」记录。

- **有记录** → 按该目标推荐
- **无记录** → 主动询问主人，主人回答后调用 `add_memory` 保存

目标与推荐原则：

| 目标 | 推荐原则 |
|------|----------|
| 增肌 | 高蛋白 + 适量碳水 + 低油 |
| 减脂 | 高蛋白 + 低碳水 + 低油 |
| 均衡 | 营养全面、比例适中 |

### Step 2: 推荐饮食方案

1. 调用 `get_weekly_summary` 检查本周已吃过的记录，避免连续重复
2. 推荐 **2 个方案**，每个方案包含：主菜 + 碳水 + 蔬菜
3. 提供方案后等待用户选择
4. 用户回复后调用 `record_meal` 记录实际选择
5. 若当日另一餐已记录，做差异化搭配

### Step 3: 周总结

按需调用 `get_weekly_summary` 获取本周饮食记录，统计维度：

- 共记录多少餐（午餐/晚餐分别计数）
- 蛋白质来源分布
- 食材多样性
- 记录覆盖率（几天有记录）

根据当前饮食目标给出优化建议。

## Templates

### 推荐方案模板

```
主人，该吃{午餐/晚餐}啦~
{如果本周已吃过某食物，注明避免重复}

推荐方案 A：
  🥩 {主菜} + 🍚 {碳水} + 🥦 {蔬菜}

推荐方案 B：
  🥩 {主菜} + 🍚 {碳水} + 🥦 {蔬菜}

选 A 还是选 B？或者有自己想吃的？😊
```

### 周总结模板

```
主人，本周饮食汇总来啦~

📅 {start} ~ {end}（第 {week} 周）
🍽️ 共记录 {total} 餐（午餐 {lunch_count} 餐，晚餐 {dinner_count} 餐）

🥩 蛋白质来源：{protein_distribution}

🥦 食材多样性：{veg_diversity}

📋 记录覆盖：{covered_days}/7 天有记录

💡 优化建议：{建议内容}
```
